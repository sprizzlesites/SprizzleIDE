How Reticle keeps your templates intact

Most browser PDF editors rasterize each page to an image, then re-encode the whole file on save. That's where your fonts get swapped for substitutes, boxes shift a few points, and crisp vector text turns fuzzy.

Reticle never touches your existing page content. It reads the original PDF object graph and copies your pages through unchanged. Anything you add — text, whiteout, images, signatures — is appended as a new content layer over the top. The bytes of your template survive.

What it does well

Editing text already in the document (click to retype), filling templates and forms, redacting/whiting-out, stamping signatures and logos, adding boxes and highlights, page reordering / rotation / deletion / blank inserts, and merging PDFs. Geometry is tracked in PDF points so placement is exact and re-applies cleanly to the same template.

Editing text already in the PDF

Select the Edit-text tool (E) and click any text. Reticle reads that text run straight from the PDF — its string, position, size and font — covers it with a patch colour-matched to the page background, and drops an editable copy in the exact same place with the font, size and colour matched. You retype; on export it reads as if you edited the original.

This is the honest browser-feasible version of in-place editing. PDFs store positioned glyphs, not paragraphs, so nothing can truly re-flow original text without re-typesetting it (which is what swaps fonts in other tools). Reticle instead reuses your typed value and never disturbs the surrounding content. Two things to know: it works one text run at a time (often a word or line), and the font is matched to the closest base font — if the match looks off, change it in the panel or embed the real font.
